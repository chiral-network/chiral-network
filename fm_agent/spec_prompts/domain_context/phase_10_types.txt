PHASE 10: TAURI COMMAND SURFACE
================================

MODULE SUMMARY
--------------
Tauri IPC command handler for the desktop GUI client. Exposes 100+ async
commands covering DHT networking, file operations, wallet/mining, hosting,
Drive/site management, and version enforcement. Commands invoke the shared
AppState (DHT, Geth, FileTransfer, etc.) and run in the Tauri async runtime.

STARTUP ASYNC TASKS
-------------------
fetch_and_log_remote_version_policy() spawned via tauri::async_runtime::spawn()
  - Fetches /api/version-policy from relay (http://130.245.173.73:8080)
  - Parses remote VersionPolicy with 8-second timeout
  - Calls version::update_effective_policy() to apply policy with validation
  - Logs acceptance/rejection with signed/unsigned status and issued_at


INVARIANTS
----------
- Every command gates through ensure_version_supported(state) before accessing
  the DHT or network; client below min_required gets Error, UI shows modal.

- Version policy is fetched asynchronously at startup; local build stays at
  bundled policy until the async task updates version::EFFECTIVE_POLICY via
  version::update_effective_policy(); RwLock gives non-blocking reads.

- File transfer operations (publish, download, send) all go through DHT
  via the shared dht::DhtService handle and command_sender channel.

- Wallet operations (send_transaction, get_balance) invoke Geth RPC via
  geth::rpc_endpoint(); mining status polls are rate-limited per UI refresh.
