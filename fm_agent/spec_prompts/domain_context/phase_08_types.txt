PHASE 8: DHT SERVICE
====================

MODULE SUMMARY
--------------
The DHT Service implements the distributed hash table layer for Chiral Network
using libp2p's Kademlia protocol (custom /chiral/kad/1.0.0). It manages:
  - File discovery and seeding via provider records
  - Key-value storage for metadata and host registry
  - P2P file transfers with chunked downloads (256 KB chunks)
  - Payment verification for paid transfers
  - Peer lifecycle management with version-policy enforcement

INVARIANTS
----------
- Chunk size is fixed at 256 KB; files are split deterministically; chunk
  hashes are SHA-256 and precomputed at registration (warming in background).

- Provider TTL is 10 minutes with a 3-minute republish cycle; offline seeders
  drop out within 10 minutes, and healthy seeders refresh ~3x before expiry.

- KV record TTL is 22 minutes with 3-minute replication and publication intervals;
  records are gated by both replication_interval and publication_interval timers.

- Signed-record verification: Identify handler stamps agent_version (with semver)
  into the peer's agent_version field; on inbound identify, peers below
  version::effective_policy().min_required trigger disconnect_peer_id().

- Payment verification for paid downloads: seeder signs chunk delivery with
  Ethereum transaction hash; downloader's wallet is checked against the
  verifyPaymentTx RPC call before subsequent chunks are sent.
