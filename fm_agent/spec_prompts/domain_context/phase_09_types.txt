PHASE 9: HTTP GATEWAY AND APIS
===============================

MODULE SUMMARY
--------------
HTTP gateway layer (Axum-based) serving Drive, Rating, Hosting, and Relay
APIs with version-policy middleware and X-Owner header authentication.
Handles: shared file uploads/downloads, reputation tracking, site proxying,
relay tunnel registration, CDN file hosting with payment verification.

version_gate_middleware(
  req: axum::extract::Request,
  next: axum::middleware::Next,
) -> axum::response::Response
  Enforces X-Chiral-Client-Version header against bundled min_required.
  Exempts: /api/version-policy, /api/health, /api/ready, non-/api/* paths.
  Returns 426 Upgrade Required if client version is below policy floor.


INVARIANTS
----------
- X-Owner header (wallet address 0x + 40 hex digits) is required on all
  mutating Drive and Rating routes; missing header returns 400 Bad Request.

- /api/version-policy endpoint is always reachable (no version gate) so
  outdated clients can discover the network's min_required and download
  newer builds; all other /api/* routes are gated by version_gate_middleware.

- CDN payment tolerance is 5% (lines 380 in cdn_server.rs):
  min_accepted_wei = required_wei * 95 / 100 for CHI→wei rounding.

- Relay share proxy uses WebSocket tunnels for NAT traversal; if tunnel is
  down, requests are proxied to origin_url; origin URL host/port are
  replaced if they are 0.0.0.0, 127.0.0.1, or localhost (fix_origin_url).

- Rating API caches event history per wallet with 90-day LOOKBACK_SECS
  (computed dynamically on every GET /api/ratings/:wallet request).
